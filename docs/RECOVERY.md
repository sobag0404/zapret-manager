# Recovery

Use this document when Zapret Manager cannot start, the service is stuck, or
connectivity is degraded after an update or profile change.

## Safe Recovery Order

1. Stop the Zapret Manager Windows service.
2. Revert to the previous known-good engine and profile if rollback metadata is
   available.
3. Restore the service startup type recorded before the failed operation.
4. Start the service only after validation passes.
5. Export diagnostics if the issue persists.

## Manual Recovery Expectations

Manual recovery commands should be generated by the application or installer so
they match the actual service name and install path. Documentation must avoid
hard-coding destructive commands.

## When to Reinstall

Use repair or reinstall only after service stop and rollback fail. Reinstall
must preserve user settings unless the user chooses a full reset.

## Full Reset

A full reset may remove Zapret Manager-owned runtime files, service
registration, profile selection, update cache, and rollback metadata. It must
not remove unrelated user files.
